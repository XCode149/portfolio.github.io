Introduction

Ce document explique, étape par étape et avec exemples réutilisables, comment concevoir un jeu point-and-click utilisant uniquement HTML, CSS et JavaScript. Le contenu est formulé de manière professionnelle et prêt à être copié-collé dans un projet.

1. Structure de base

Utilisez des <div> modulaires pour séparer la logique et l'affichage : une « scène » (conteneur), un panneau d'inventaire, une zone de dialogues et des hotspots positionnés en absolu.

<div id="app" class="game">
  <div id="scene" class="scene"></div>
  <div id="ui" class="ui">
    <div id="inventory" class="inventory"></div>
    <div id="dialogue" class="dialogue"></div>
  </div>
</div>

Avantage : tout est contenu dans des <div>, facile à styliser et à rendre réutilisable.

2. Hotspots (zones cliquables)

Créez des éléments positionnés absolument et contrôlez-les via JavaScript. Exemple de style réutilisable :

.hotspot{position:absolute;cursor:pointer;outline:2px solid transparent}
.hotspot:hover{outline-color:var(--accent)}

Gérez les hotspots via un tableau d'objets (x, y, largeur, hauteur, action).

3. Gestion des scènes

Stockez les scènes dans un objet JS et chargez dynamiquement le HTML/CSS correspondant. Exemple d'API minimale :

const scenes = {
  'room1':{bg:'room1.jpg',hotspots:[/*...*/]},
  'room2':{bg:'room2.jpg',hotspots:[/*...*/]}
}
function loadScene(name){/* met à jour #scene */}

4. Inventaire

Représentez l'inventaire par un tableau d'objets et rendez-le dans un <div>. Exemple :

const inventory = [];
function pickItem(item){inventory.push(item);renderInventory()}

Offrez la possibilité de configurer l'affichage (taille d'icônes, comportement au clic) via des classes CSS.

5. Dialogues

Structure simple : tableau de lignes et contrôleur d'affichage. Séparez données et rendu pour pouvoir traduire facilement.

function showDialogue(lines){/* affiche une boîte et boucle sur lines */}

6. Mini-moteur

Centralisez la logique : gestionnaire d'état, loader de scènes, système d'événements, inventaire, et sauvegarde (localStorage). Exemple d'API :

const Engine = {
  state:{scene:null,inventory:[]},
  loadScene(name){/*...*/},
  triggerHotspot(id){/*...*/}
}

7. Interface utilisateur (personnalisable)

Le design est construit autour de classes: .scene, .hotspot, .inventory, .dialogue. Modifiez les variables CSS en haut du document pour adapter le rendu (couleurs, rayon, ombre).

/* Exemple : personnaliser via :root */
:root{--accent:#c9a77a;--paper:#fffdf8;--radius:12px}

8. Apprendre le JavaScript (par étapes pratiques)

  1. Notions de base : variables, fonctions, structures de contrôle.
  2. Manipulation du DOM : sélectionner, créer et modifier des <div>.
  3. Événements : gérer les clics, mouvements et entrées clavier.
  4. Sérialisation et stockage : sauvegarder l'état via localStorage.
  5. Modularité : séparer utilitaires, scènes et UI dans des fichiers ou modules.
  6. Mise en pratique : implémentez un mini-moteur et améliorez-le progressivement.

Ressources recommandées : MDN, tutoriels JS modernes, et travaux pratiques.
